Sistema de Ficheros: Linux 

Todos los sistemas de ficheros de Linux implementan un conjunto básico de conceptos comunes derivados del sistema operativo Unix.
Los ficheros están representados por inodes, los directorios son simplemente ficheros que contienen una lista de entradas 
y se puede acceder a los dispositivos solicitando E / S en ficheros especiales.

Inodes

 Cada fichero está representado por una estructura, llamada inode.
Cada inodo contiene la descripción del fichero:
tipo de fichero, derechos de acceso, propietarios, marcas de tiempo, tamaño, punteros a bloques de datos. 
Las direcciones de los bloques de datos asignados a un fichero se almacenan en su inodo.
Cuando un usuario solicita una operación de E / S en el fichero, el código del núcleo convierte el desplazamiento actual en un número de bloque, 
usa este número como índice en la tabla de direcciones de bloque y lee o escribe el bloque físico.

 El sistema de archivos o sistema de ficheros es el componente del sistema operativo encargado de administrar.​ 
Sus principales funciones son la asignación de espacio a los archivos, la administración del espacio libre y del acceso a los datos resguardados.
Además se implemento uno de estos.
La mayoría de los sistemas operativos manejan su propio sistema de archivos:

* LINUX Etx2 , Ext3 , etc (Extended file system).
* SOLARIS ZFS (Zettabyte File System). 
* Mac OS HFS+(Hierarchical File System Plus).
* Windows NTFS ( New Technology File System).
En Linux y Unix todo es un fichero.Los directorios son ficheros, los ficheros son ficheros, y los dispositivos son ficheros.
A veces a los dispositivos se les llama nodos, pero siguen siendo ficheros.
Los sistemas de ficheros de Linux y Unix se organizan en una estructura jerárquica, de tipo árbol.
El nivel más alto del sistema de ficheros es / o directorio raíz.
Todos los demás ficheros y directorios están bajo el directorio raíz.
/The Root Directory » Todo el sistema Linux se encuentra en el directorio /, conocido como el directorio raíz.
/bin » contiene los binarios (programas) esenciales del usuario que deben estar presentes cuando el sistema se monta en modo de usuario único.Aplicaciones
/boot » contiene los ficheros necesarios para arrancar el sistema, Scripts o directorios de scripts ; por ejemplo, los ficheros del cargador de arranque y los núcleos de Linux.
/dev » contiene ficheros especiales que representan dispositivos. Estos no son ficheros , se muestran como ficheros; ejemplo: /dev /sda.
/etc » contiene ficheros de configuración, generalmente se pueden editar.
/home » contiene una carpeta de inicio para cada usuario y los ficheros de datos del mismo.
/lib » contiene librerías compartidas necesarias para los programas que se encuentran en el sistema de ficheros raíz./bin y /sbin.
/media » contiene subdirectorios donde se montan los dispositivos de medios extraíbles insertados.
/mnt » contiene puntos de montaje temporales,ra los sistemas de ficheros montados por el administrador del sistema.
/opt » contiene subdirectorios para paquetes de software opcionales , donde instalar aplicaciones opcionales (de terceros).
/proc » directorio dinámico especial que mantiene información sobre el estado del sistema, incluyendo los procesos actualmente en ejecución
/root » directorio personal del usuario root (superusuario);
/run »contiene ficheros transitorios para procesos.
/sbin »contiene binarios de administración del sistema ,destinados a ser ejecutados por el usuario root para la administración del sistema.
/srv »contiene datos para los servicios proporcionados por el sistema. HTTP ,NTTP , entre otros.
/sys »contiene contiene ficheros que proveen información del kernel a eventos del sistema operativo.
/tmp » contiene ficheros temporales Las aplicaciones almacenan estos.Generalmente se eliminan cada reinicia el sistema.
/usr » contiene aplicaciones y ficheros a los que puede acceder la mayoría de los usuarios
/var »contiene ficheros variables como ficheros de registros y bases de datos.
